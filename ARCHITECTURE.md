# 项目架构设计

## 架构目标

实现一个完整的项目创建、沙箱测试和部署流程，确保新项目能够在测试成功后运行，测试失败后打回重写。

## 核心组件

### 1. 配置管理模块 (`config/`)
- 负责管理不同环境的配置
- 支持多AI模型配置
- 提供统一的配置访问接口

### 2. 项目创建模块 (`project/`)
- 负责根据用户需求生成新的项目结构
- 支持生成完整的项目文件和目录
- 提供项目模板管理功能

### 3. 沙箱管理模块 (`sandbox/`)
- 负责创建和管理隔离的测试环境
- 支持环境的创建、销毁和重置
- 提供文件同步和隔离功能

### 4. 测试评估模块 (`testing/`)
- 负责执行项目测试
- 支持自动测试和手动测试
- 提供测试结果分析和评估

### 5. 部署模块 (`deployment/`)
- 负责将测试通过的项目部署到目标环境
- 支持部署前的准备和部署后的验证
- 提供回滚机制

### 6. 主控制器 (`controller/`)
- 负责协调各个模块的工作
- 实现完整的工作流程
- 提供用户交互接口

## 工作流程

1. **项目创建**：用户输入需求，项目创建模块生成项目结构
2. **沙箱准备**：沙箱管理模块创建隔离的测试环境
3. **项目测试**：测试评估模块在沙箱中执行测试
4. **结果评估**：测试评估模块分析测试结果
5. **部署决策**：根据测试结果决定是否部署
6. **项目部署**：部署模块将测试通过的项目部署到目标环境
7. **部署验证**：验证部署是否成功

```
│   ├── __init__.py
│   └── settings.py
├── project/               # 项目创建
│   ├── __init__.py
│   └── generator.py
├── sandbox/               # 沙箱管理
│   ├── __init__.py
│   └── manager.py
├── testing/               # 测试评估
│   ├── __init__.py
│   └── evaluator.py
├── deployment/            # 部署模块
│   ├── __init__.py
│   └── manager.py
├── controller/            # 主控制器
│   ├── __init__.py
│   └── main.py
├── utils/                 # 工具函数
│   ├── __init__.py
│   └── helpers.py
├── prompts.py             # AI提示词
├── main.py                # 主入口
├── requirements.txt       # 依赖
└── .env.example           # 环境变量模板
```
